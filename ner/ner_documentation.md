# 📑 Named Entity Recognition (NER) Module Documentation

## 🔄 Module Overview
The NER module serves as the core semantic processing layer for the **AI Contract Intelligence Pipeline**. It accepts unformatted text data generated by the OCR module, filters out structural noise, isolates business entities, and serializes the findings into a flat JSON format.

## 🎯 Targeted Entity Classifications
To fulfill contract metadata extraction goals, the module targets three critical keys:
1. **`ORG` (Organization Names):** Identifies contracting corporate bodies, institutions, and legal entities.
2. **`DATE` (Contract Milestones):** Captures execution dates, effective periods, and deadlines.
3. **`MONEY` (Financial Valuation):** Extracts compensation limits, breach liabilities, and contractual fees across multiple currency markers (`$`, `₹`, `€`, `£`, `USD`, `INR`).

## 🛠️ Multi-Layered Accuracy Architecture
To maximize production extraction precision, the engine operates on a hybrid pipeline:
* **Layer 1 (Statistical AI Model):** Utilizes spaCy's `en_core_web_sm` transformer token model to run semantic context checks.
* **Layer 2 (Heuristic Sorting):** Cross-references misclassified strings (e.g., organizations containing currency symbols are dynamically reassigned to financial data lists).
* **Layer 3 (Deterministic Regex Fallback):** Runs custom regex sweeps to catch complex formatting anomalies often created by OCR systems (such as localized Indian numbering formats like `₹8,50,000`).

## 1. Localized Bias Mitigation & Architectural Enhancements
During empirical pipeline validation using mixed regional contract samples, structural information leakages were identified within standard Western pre-trained models (`en_core_web_sm`). Specifically, Indian geopolitical sub-states (e.g., *Tamil Nadu*) and country-specific financial compensation tokens (e.g., *4LPA*) were bypassed or misclassified by the baseline statistical model weights.

### 🛠️ Resolution Architecture Implemented:
* **Geographical Gazetteer Layer:** Structured an explicit, deterministic lookup boundary matrix to parse and map Indian sub-territories seamlessly.
* **Alphanumeric Salary Regex Compilers:** Overlaid a customized regular expression tokenizer targeting macro financial boundaries (`\d+\s*(?:LPA|Lakhs?|Cr)`).
* **Production Post-Processing Re-Router:** Built an automated cross-entity safety loop to catch currency strings (like `INR 5,00,000`) that statistically leak into the `ORG` class, re-routing them to the `MONEY` payload array before server compilation.
